原版 WorkBuddy 的记忆机制

原版只有三层：会话级 + 任务级 + 文件级

层级 存储方式 生命周期 可控性
对话历史 服务端自动保存，云端存储 任务会话内连续，会话结束消失 用户无法直接查看或编辑
任务上下文 服务端维护，任务级隔离 新建任务需重新描述背景 系统自动管理
文件上下文 主动上传或拖拽文件 随任务结束清除 每次手动关联
"原版 WorkBuddy 通过「保留工作空间」功能实现类似记忆的效果——在同一个项目中持续追问时，减少重复切换。但本质上，这只是一次性的会话缓存，没有持久化结构。"

原版核心局限

  • 会话结束 → 上下文彻底消失，跨会话零连续性
  • 无 MEMORY.md → 没有用户可编辑的长期记忆文件
  • 多工作空间各自为政，无法跨空间共享上下文
  • 子代理输出直接进入主上下文，Token 爆炸
  • 无操作日志 → 无审计追溯，决策黑箱

自定义系统的六层连续性架构
L6
专项记忆 *.md
关键词触发，按需加载（文章列表、工具说明、项目档案）
L5
Rolling Summary
会话结束自动生成结构化摘要，新会话优先读摘要跳过完整历史
L4
Cache Sidechain
文件 mtime 缓存，读取前查 mtime 一致则跳过，避免重复加载
L3
Sidechain Transcript
所有写操作统一追加：FILE / API / DECISION / NOTE / SUMMARY
L2
MEMORY.md
核心常驻，每次会话加载：工作约定、品牌规范、技术约束、关键结论
L1
Agent Identity
身份摘要（SOUL.md / IDENTITY.md / USER.md），每次会话启动

参考来源：Claude Code 四条不变量 + 七层控制面

2026-04-07 全面升级时，参照了 Claude Code 源码分析提炼的架构设计。Claude Code 通过四条不变量（身份、记忆、操作链、内容状态）和七层控制面（I/A/B/C/D/E/F/G）实现了跨会话的连续性，而自定义系统将这些思想适配到了 WorkBuddy 环境中，演化为更精简的六层架构。

自定义系统五大核心协议

单一写入协议（Single Write Protocol）

所有操作统一写入 sidechain_transcript.md，不再分散维护 cache_sidechain.md 和 daily log 写入。格式为 [类型] 时间戳 | 操作 | 目标 | 结果，派生规则从 sidechain 统一派生视图层和日报层。

多工作空间记忆扫描协议

扫描 4 个目录：Claw（主）/ qclaw / openclaw / mac_mini。同一日期数据合并而非覆盖，合并任务列表、技能调用、文章产出。场景：生成工作日志报告时全量合并。

每日记忆同步协议

每个新任务当日 .workbuddy/memory/YYYY-MM-DD.md 更新后，必须同步追加到主工作空间 Claw 的对应日期文件（同一日期汇总，不是覆盖）。漏记补录：发现历史某日缺失日志，找到对应工作内容后补写。

文章文件唯一存储协议

文件从不进 WorkBuddy 任务目录。文章所有文件只存在于 /Volumes/mac_mini/龙虾/{文章名}/，包含 HTML、配图、研究素材 .md、原始数据。发布后保留内容，清理任务目录中间文件。

会话启动协议

第一步：扫描 sidechain_transcript.md 末尾 ##UNFINISHED## 区域，如有未完成任务开场时告知用户。第二步：检查记忆裁剪，超过阈值触发自动归档。第三步：确认续接意愿，展示上次未完成 + 最近会话摘要 + 是否有新任务。

自定义系统的五大核心好处
01

真正的跨会话连续性

原版 WorkBuddy 每个任务会话结束后一切归零。自定义系统通过 MEMORY.md 和每日日志，实现了真正的跨会话连续——上周做的决定，下周还能追溯。

02

Token 效率革命性提升

通过 Cache Sidechain（mtime 缓存跳过）、Rolling Summary（跳过完整历史）、Subagent 隔离（raw→summary 降级）三层机制，长会话 Token 消耗预估下降 40-60%，实测 Harness Engineering 场景提升 73%。

03

多工作空间统一管理

主工作空间 Claw + qclaw + openclaw + mac_mini 四个空间并行运作，通过统一扫描协议合并数据，一次对话可以调动所有空间的上下文。

04

操作完全可审计追溯

Sidechain Transcript 让每一次文件操作、API 调用、关键决策都有时间戳记录。不是黑箱——用户可以回溯任何一次决策的依据和过程。

05

用户拥有完全控制权

所有记忆都是手写的 .md 文件。用户可以随时查看、修改、补充。不依赖系统自动生成，记忆内容完全真实、可信、可控。

总结

原版 WorkBuddy 提供的是「会话级即时记忆」——高效但短暂，适合单次任务。自定义记忆系统在此基础上叠加了「持久化层」：MEMORY.md 的核心常驻让每次会话都有稳固的上下文基座，Sidechain Transcript 的操作日志让所有决策有据可查，Rolling Summary 的会话摘要让跨会话跳转成为可能，Memory Pruning 防止文件无限膨胀。

核心哲学只有一条：把 AI 的记忆变成用户可以查看、编辑、信任的文件。不是交给系统黑箱，而是握在自己手里。
